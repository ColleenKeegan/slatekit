Slate Kit

Overview

A Scala toolkit, utility library and server backend.

About

Slate Kit is a tool kit, utility library, and server backend for mobile and web apps. It is written entirely in Scala, uses Akka-Http for the Server, and works with AWS and MySql. Slate Kit is currently used to build the backend architecture for a few mobile beta apps.. We have taken a simple and pragmatic approach to building the tool-kit. It is designed for simplicity, practicality and code maintainability.


Simple & Practical



Modules

Slate Kit is modular and contains many reusable components ( see list below ). One of the main design philosophies has been to build protocol independent API's that can be "embedded" into a Web API Server or "embedded" into a CLI (Command Line Interface) Shell.

Component About Description
1. App Base Application A powerful base app with support for command line args, environment selection, configs per environment, logging, life-cycle events, encryption, diagnostics and more
2. APIs Protocol Independent APIs Easily build protocol independent APIs using simple scala classes/methods with annotations, that can be hosted / run on both Slate Command Line Shell and/or as Web APIs in the Slate Server
3. CLI Command Line Interface A command line interface shell that you can extend to host your "Protocol Independent APIs" and/or hook into to handle and execute any user commands in a shell.
4. Server Web API Server A Web API server built using Akka Http that can host your "Protocol Independent APIs". You can extend this server and register your own APIs.
5. ORM Domain-Driven ORM A simple, light-weight, Domain-Driven, single table ORM to map your Scala entities to and from database tables. MySql is currently supported with support for MS Sql Server coming soon.
6. Architecture Architecture Components Architectural abstractions and implementations for APIs, CLI, Files, Queues, Tasks and more. Files and Queues are easy to use abstractions over AWS S3 and SQS.
7. Utilities Utility Components Many useful utility components and code that can be used for any application. All these are located in the Slate.Common project and independent and modular.
8. Mobile Mobile / Web Features Mobile and web features for most applications, such as Users, Devices, Registration, Invites, Settings and more.

Philosophy

Simplicity above all else


Tech 100% Scala components, utilities, server code.
System Designed for MySql, AWS ( S3, SQS ), Akka-Http.
Design This is designed as a modular Tool-Kit via a collection of project libraries containing components.
Use as little or as much as you need.
Approach Simplicity and Practicality above all else
Modular Composable, single-purpose components as building blocks. Organized into various projects.
Portable Designed for low vendor lock-in via a "library" based approach.
This is NOT a framework
Minimal Dependencies on external, 3rd-party libraries are kept to a minimum ( many are optional )
Style We aim for immutable data and functional programming as much as possible.
We use many Scala language features that we believe are incredibly powerful and yield concise code.
However, this is NOT a 100% purely functional tool-kit.
We also carefully avoid Scala features in specific times/places in favor of simple, understandable, maintainable code.
Conventions Convention over configuration
Open Source Hybrid Open-Source. All of Slate Kit is open-source except for the Slate.Ext project containing the mobile / web features
License Apache 2.0 : Please check our git repo for more info
