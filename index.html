Slate

Slate Kit is a simple, and very useful utility library and micro-framework for Scala. It provides the technical foundation to quickly build Scala apps. Save time and effort on technical infrastructure so you can focus on building the business features of your product or start-up. Slate is built on the concept of using modular, re-usable code and components to get your apps developed, and rolled out to users as quickly as possible.

Goals

Slate solves some very common problems in the technology area:

   
Rapidly create scala apps and prototypes
Reduce costs of development
Focus on business features, not technology needs
Well documented apis and components
Support for cloud services ( Amazon S3, SQS )
Request features, log issues, and contact support

Modules

Overview

Modules Description
Args A lexical arguments/options parser for command line parsing and more
Auth Simple authorization component for both desktop and concurrency based apps
S3 A light-weight abstraction over file storage with support for Amazon S3
SQS A light-weight abstraction over messaging with support Amazon SQS
Api A mini api container to easily built and host protocol independent apis that can be used in the web or command line
App A robust application template with built-in support for command line args, environments, logging, configs and more
Commands A robust implementation of the command pattern
Csv A csv lexical parser for reading any type of csv files
Database A collection of database related functions
Entities A micro-framework and ORM for managing your data models
Lex A lexical parser that can be extended
Logging A simple logger that can be extended with 3rd party logging software
Model A component to easily build up schema for some data model
Mapper Maps data from 1 source to another source
Query Construct your own query object model for handling queries
Reflector Convenience wrapper around reflection in Scala
Shell A command line shell for your services
Tasks A component to easily build batch, background or continuously running jobs
Utils More than a dozen plus utilities and more

example Example_Args 
    // Use case 1: Defaults e..g prefix = "-", sep = ":"
    showResults( Args.parse("-env:dev -text:'hello word' -batch:10") )

    // Use case 2: Custom prefix and sep e.g. "!" and separator "="
    showResults( Args.parse("!env=dev !text='hello word' !batch=10 ", prefix = "!", sep = "=") )

    // Use case 3a: Check for method call in the beginning
    showResults( Args.parse("service.method -env=dev -text='hello word' -batch=10", prefix = "-", sep = "=", hasAction = true ) )

    // Use case 3b: Check for method call in the beginning
    showResults( Args.parse("method -env=dev -text='hello word' -batch=10", prefix = "-", sep = "=", hasAction = true ) )

    // Use case 4: No args
    showResults( Args.parse("service.method", prefix = "-", sep = "=", hasAction = true ) )

    // Use case 5a: Help request ( "?", "help")
    showResults( Args.parse("?"         ) )

    // Use case 5b: Help request with method call ( "method.action" ? )
    showResults( Args.parse("service.method help"   , hasAction = true ) )

    // Use case 6: Version request ( "ver", "version" )
    showResults( Args.parse("version"  ) )

    // Use case 7: Exit request
    showResults( Args.parse("exit"     ) )

    // Use case 8: apply args to an object
    Args.apply("-env:dev -text:'hello word' -batch:10", new SampleOptions(), "-", ":", true)

example Example_Auth 
    // Setup 1: For web / con-current apps ( use separate instance of authWeb per user )
    val user1 = new User( "1", "user 001", "user", "001", "user001@gmail.com", "123-456-7001", false, false, true)
    val authWeb = new AuthWeb().init(isAuthenticated = true, user = user1, roles = "moderator")

    // Setup 2: For local / desktop ( non-concurrent apps - use 1 instance of AuthDesktop )
    // and initialize the singleton Auth with the desktop implementation for convenient access
    val user2 = new User( "2", "john doe", "john", "doe", "jdoe@gmail.com", "123-456-7890", false, false, true)
    Auth.init(isAuthenticated = true, user = user2, roles = "admin")

    // Use case 1: Use the authWeb ( per user / concurrent service ) to check auth info
    println ("Checking auth info in web/concurrent mode" )
    println ( "user info         : " + authWeb.user                   )
    println ( "user id           : " + authWeb.userId                 )
    println ( "is authenticated  : " + authWeb.isAuthenticated        )
    println ( "is email verified : " + authWeb.isEmailVerified        )
    println ( "is phone verified : " + authWeb.isPhoneVerified        )
    println ( "is a moderator    : " + authWeb.isInRole( "moderator") )
    println ( "is an admin       : " + authWeb.isInRole( "admin" )    )
    println ()

    // Use case 2: Use singleton Auth class for desktop apps to conveniently access auth information
    println ("Checking auth info in desktop/local mode" )
    println ( "user info         : " + Auth.user                   )
    println ( "user id           : " + Auth.userId                 )
    println ( "is authenticated  : " + Auth.isAuthenticated        )
    println ( "is email verified : " + Auth.isEmailVerified        )
    println ( "is phone verified : " + Auth.isPhoneVerified        )
    println ( "is a moderator    : " + Auth.isInRole( "moderator") )
