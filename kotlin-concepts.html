Key concepts in Slate Kit

Overview

There are just a few key concepts in Slate Kit that you need to know to get up and running quickly.


Concept About More
1. Env Represents a logical environment such as local, dev, qa, prod more
2. Config Configuration settings as a thin layer on Java Properties with extra features. There is 1 config file per environment more
3. Context The AppContext is a container for common dependencies ( env, config, logger, encryptor ) and is passed to components. more
4. Apis APIs in Slate Kit are are normal methods on classes and are "protocol independent" which means they can be made accessible on the CLI ( command line ) or as an HTTP Endpoint more
5. Cloud Slate Kit integrates with Cloud infrastructure such as File Storage and Queues. AWS ( S3, SQS ) is the default but its abstracted so you can easily customize to other providers. more
6. Request Abstracts a request whether it is on the CLI ( command line ) or a HTTP Request. This is used to build Slate Kit APIs ( which are protocol independent ). more
7. Result Abstracts a response and models success/failures, whether it's on the CLI (Command Line) or an HTTP Response. This is used to build Slate Kit APIs and is compatible with HTTP status codes more


1. Envs

You can setup different environments ( local, dev, qa1, qa2, etc ) for your application. This is accomplished using the Env component below. It is used in the Slate Kit App ( Base App )

Features

  1. Validates the selected env at startup
  2. Describe the selected env
  3. Store selected environment from available ones
  4. Naming convention to link to config files ( below )
  5. Customize the environments supported

2. Configs

All the different types of Apps and Sample Apps using a common configuration scheme. For each app, there is a single common configuration file env.conf and then 1 environment specific configuration file for each environment. e.g. env.dev.conf All the environment configs are located in the resources directory but you can put them and reference them anywhere. Slate Kit configs are simple abstractions over Java properties files. It is used in the Slate Kit App ( Base App )

Features

  1. Typed settings ( bool, string, short, int, long, float, double, date )
  2. Lists of basic types
  3. Maps of basic types
  4. Default values if keys do not exist
  5. Decryption for encrypted settings
  6. Inheritance via loading a common config
  7. Auto-mapping of custom objects (Db connections, Api-keys)
  8. URI references to other config files/settings

3. Context

The context is a container for global objects needed for most components. Such items include the current environment ( dev, qa, pro, etc ), config settings, a logger, database info, encryptor, the host computer, the runtime version of java, and access to the Entities that provide the ORM entity services.
It is created at startup and is read-only.

Features

  1. Validates the selected env at startup
  2. Describe the selected env
  3. Store selected environment from available ones
  4. Naming convention to link to config files ( below )
  5. Customize the environments supported
